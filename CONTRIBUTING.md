# Contributing

Contributions should keep the project defensive, legal, and beginner-friendly.

Accepted contribution types:

- New sample logs
- New defensive detections
- New blue-team labs
- Documentation improvements
- Dashboard placeholder improvements
- Bug fixes for scripts or Docker services

Not accepted:

- Malware
- Credential theft tooling
- Persistence tooling
- Exploit code against real systems
- Instructions for attacking systems outside an authorized local lab
- Destructive payloads

Before submitting a pull request:

1. Run the local detections.
2. Confirm logs are synthetic or sanitized.
3. Update the README if structure changes.
4. Keep all examples focused on authorized local training.
