# Example Incident Report Summary

This is a short example only. Learners should complete their own report using `REPORT_TEMPLATE.md` during Lab 10.

## Summary

The sample logs show repeated failed SSH login events, suspicious web request patterns, a suspicious shell-like process event, an unusual network destination port, and a sensitive file change. These events should be correlated into a timeline and reviewed as a simulated incident.

## Training Note

Do not treat this as a real incident. The data is synthetic and exists only for local blue-team practice.
