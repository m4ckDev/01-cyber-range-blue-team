# Local-Only Safety Rules

This cyber range is designed for safe, authorized blue-team practice.

## Allowed

- Running the Docker lab locally
- Reviewing sample logs
- Running detection scripts against included logs
- Writing reports from simulated evidence
- Modifying the sample data for personal learning
- Building defensive detection logic

## Not Allowed

- Attacking real systems
- Scanning systems that are not yours
- Using this project to collect credentials
- Adding malware or persistence code
- Adding real private logs or secrets
- Running tests against networks without written permission

## Why the Lab Uses Sample Data

The goal is to learn investigation and detection without harming systems or crossing legal boundaries. The included logs are synthetic and intentionally safe.

## Analyst Mindset

Treat the lab like a real investigation, but remember:

- Evidence must come from logs.
- Do not assume impact without proof.
- Separate facts from hypotheses.
- Document every conclusion.
