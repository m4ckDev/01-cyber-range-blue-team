# Security Policy

This repository is a defensive training project. It should only include safe sample data, simulated events, and defensive code.

## Reporting a Concern

Open an issue if you find:

- Unsafe content
- Real credentials
- Real private logs
- Offensive instructions
- Broken lab logic
- A detection script that behaves unexpectedly

## Project Boundaries

This project must not include:

- Real malware
- Credential theft code
- Persistence mechanisms
- Exploit chains
- Instructions to attack real systems
- Destructive commands

All training activity should remain local, authorized, and defensive.
